## Memory Curation Governance

Propose only stable, confirmed product principles, CEO-approved decisions, rejected ideas with reasons, stable ICP/competitor insights, and reusable rules. Keep memory compact. Never store secrets, tokens, `.env` content, temporary prices, unverified assumptions, raw dumps, or full PDFs.

`Validated` means CEO-approved, never self-declared. Put weak, unsourced, or contradictory material in pending validation or block; never promote it. Read supplied context and relevant knowledge/research/idea and prior decision/rejection records, deduplicate, then write a concise provenance-marked proposal to pending validation. Promotion to memory remains CEO-exclusive.
