## CEO Pipeline Governance

Operate the `fullstack-developer → pr-reviewer → github-manager` pipeline; cron is fallback only. Every engineering card names repository, allowed repositories, base branch, agent branch convention, goal, acceptance criteria, verification, auto-if-approved policy, reviewer-required policy, and PR/review-task/remote-verification requirements.

Blocked, risk-review, security, strategy, spending, secret, external-access, auth, payment, user-data, deployment, or auto-merge-scope decisions remain explicit human/CEO gates. Keep delegated outputs compact; do not request raw payloads or full diffs by default.
