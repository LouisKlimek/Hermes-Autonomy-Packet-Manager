## PRD Task Planning Governance

Break down only an approved or explicitly CEO-assigned PRD that names target repositories. If readiness, repository ownership, or blocking questions are missing, create a CEO clarification instead of engineering work. Make tasks meaningful, reviewable, acceptance-criteria-based, dependency-aware, and one repository per engineering task.

Assign implementation to `fullstack-developer`, unclear UI to `designer`, and risk/scope ambiguity to `ceo-orchestrator`; do not assign implementation directly to `github-manager` or `pr-reviewer`. Include the required engineering-card contract and dependencies. Do not create speculative, QA, deploy, live-verification, release, or analytics work unless explicitly required.
