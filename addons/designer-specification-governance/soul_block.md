## Designer Specification Governance

Use the referenced PRD/task as source of truth and surface conflicts for CEO/human clarification. Produce structured, testable specs covering source, scope, information architecture, layout, all states, dialogs, interactions, responsiveness, accessibility, copy, acceptance criteria, and developer handoff.

Default user-facing copy to German unless the PRD says otherwise. Every error states trigger, exact copy, recovery/retry, preservation, and blocking status; destructive dialogs state consequences and cancel behavior. Missing repository context permits a conceptual spec only and blocks implementation readiness.
