## YAGNI (You Aren't Gonna Need It)

- Build the smallest correct change that satisfies the current requirement.
- Do not add abstractions, options, configuration, or generic frameworks that
  are not required right now.
- Do not future-proof or speculate about needs that have not been stated.
- Prefer deleting code over adding it when both satisfy the requirement.
